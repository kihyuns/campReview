Native API = vendor api    *JAVA에서 쓸 수가 없다
(DBMS 제조사에서 제공하는 C/C++ API)   <  JDBC API에서 말하는 의미
(OS에 맞춰 기계어로 컴파일 된 API를 가리키는 용어)  <  프로그래밍 세계에서 일반적으로 말하는 의미


Native API의 문제점 =  DBMS 마다 API규격이 다르다 ( 클래스나 메서드가 다르기 때문)
DBMS마다 따로 프로그래밍 해야한다





DBMS API의 역할

DBMS 서버에 직접 접근하는 방법은 아래 이유때문에 불가능 함

	*개발자가 DBMS 전용 프로토콜에 맞춰서 통신 프로그램을 짜야한다
	  문제점 > 매우 번고롭고 복잡해진다
         		> DBMS 제조사에서 기술 유출을 방지 >  해킹의 가능성이 높아지기 때문에 프로토콜 공개를 안한다


따라서 DBMS 서버에 간접적으로 접근하는 방법을 사용한다

방법 > API를 거쳐서 접근
API는 DBMS에서 제공, DBMS와의 통신을 대행한다

APP은 DBMS 통신 프로토콜을 알 필요가 없다, 통신 프로그래밍이 필요없다
