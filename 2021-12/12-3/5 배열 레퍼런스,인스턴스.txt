배열 - 같은 종류의 메모리를 쉽게 만드는 방법

int i1, i2, i3, i4, i5

> int[] arr1 = new int[5];

배열의 크기는 int 타입의 최대 값에서 2를 뺀 값이다
int[] arr3 = new int[2147483645] 
int[] arr3 = new int[Integer.MAX_VALUE - 2] (계산이 힘들어서 이 명령어를 쓴다)



*배열 정리
레퍼런스(reference) - 값이 아닌 메모리의 주소를 담는 변수

인스턴스(instance) - 값을 저장하는 메모리


배열 레퍼런스와 배열 인스턴스 

*언제든 배열 인스턴스의 주소를 다른 레퍼런스에 담을 수 있다
ex) int[] arr2 = arr1  -> arr2와 arr1은 같은 배열 인스턴스를 가리킨다 ex) arr2[0] = 100

-arr2를 통해 값을 저장한 후, arr1을 통해 값을 꺼낼 수 있다


*배열 레퍼런스를 초기화시키고 싶다면 null로 설정하라
-숫자 0을 넣으면 안된다

배열 레퍼런스가 초기화 되지만 배열 인스턴스는 초기화할 수 없다


*Garbage
배열 레퍼런스의 주소를 바꾸게 되면 주소를 잃어버려 더이상 쓸 수 없는 메모리를 garbage라 부른다

garbage가 쌓이고 JVM이 종료하면 JVM에게 줬던 메모리를 운영체제가 회수한다
*JVM이 종료되는 순간 JVM이 사용했던 모든 메모리는 운영체제가 강제로 회수한다


메모리가 부족하거나 cpu가 한가할 때 Garbage Collector가 가비지를 제거한다
